## 一点五、工程规范（最佳实践）  ⚠️ 全局强制

> 从阶段 1 起，所有代码必须遵守以下原则。违反者重构。

### 核心原则

| 原则 | 含义 | 在本项目中的体现 |
|---|---|---|
| **组合优于继承** | 通过组合节点/组件构建行为，而非深继承链 | Customer = Sprite2D + Area2D + 状态脚本，不继承"EntityBase" |
| **低耦合** | 模块间依赖尽可能少、通过信号/接口通信 | Customer 只发 `order_completed` 信号，不直接调用 GameManager |
| **高内聚** | 每个模块只做一件事 | GameManager 管货币 / Customer 管交互 / HUD 管 UI / Main 负责编排 |
| **单一职责 (SRP)** | 一个类/脚本只有一个变更理由 | 不在 Customer 里写 HUD 更新逻辑 |
| **依赖倒置 (DI)** | 高层模块不依赖低层实现，依赖抽象 | Spawner 通过参数注入 spawn 目标节点，而非硬编码路径 |
| **开闭原则 (OCP)** | 对扩展开放，对修改关闭 | 新增顾客类型只需加 JSON + 新场景，不改现有代码 |

### Godot 特定实践

- **信号优先于直接引用**：跨节点通信一律用 `signal` + `connect()`，避免 `get_node("...")` 拿外部引用
- **autoload 最小化**：GameManager 是唯一 autoload，仅持有全局状态；业务逻辑不在 autoload 中
- **数据驱动配置**：游戏数值（金币奖励、生成间隔等）可从 `data/*.json` 加载或暴露为 `@export` 常量
- **场景树即架构图**：父子关系 = 组合关系；信号连接 = 发布-订阅
- **避免全局状态污染**：除 GameManager 的货币字段外，其余状态封装在各自节点内

## 四、Godot 4 关键配置片段

**project.godot（窗口）**
```ini
[display]
window/size/viewport_width=680
window/size/viewport_height=420
window/per_pixel_transparency/allowed=true
window/per_pixel_transparency/enabled=true
```

**main.gd（无边框透明）**
```gdscript
extends Node

func _ready() -> void:
    var win := get_window()
    win.mode = Window.MODE_WINDOWED
    win.borderless = true
    win.transparent = true
    win.always_on_top = false
    win.position = Vector2i(1280, 720)
```

## 七、关键经验 / 备注

- Godot 4 的 `transparent=true` 必须配合 `per_pixel_transparency/allowed=true`，否则只是硬切到窗口背景色
- 放置游戏的存档核心是"最后离线时间"字段，用于离线收益计算
- 存档路径用 `user://` 才能挂上 Steam Cloud
- 早期就留好 Steam 成就钩子，避免后期返工
