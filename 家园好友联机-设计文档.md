# 家园好友联机 · 设计文档

> 目标：在家园界面接入 Steam 真人好友联机，好友化身可停留在家园中聊天、并做出各自的动作（坐椅子 / 站收银台 / 看展架）。
> 参考：Bongo Cat「Meowtiplayer」（2025-08）——无自建服务器、Steam 大厅 + Valve 中继、仅同步表现层。
> 最后更新：2026-07-23

---

## 一、核心结论（先看这里）

1. **联机走 Steam 原生框架，不自建服务器。** 用 GodotSteam + `SteamMultiplayerPeer`，数据全经 Steam 大厅 + Valve 中继，免端口转发、免 NAT 穿透配置。这就是 Bongo Cat「无专用服务器 / 最多 100 人」的真相：房主当 host，Valve 中继兜底网络。
2. **只同步表现层，绝不同步经济进度。** 同步化身的动作意图、位置、聊天气泡；**不同步**金币 / 库存 / 展架 / 农场。这是 Bongo Cat 成功的关键，也是防作弊与一致性的根本。
3. **动作用「意图同步 + 本地锚点投射」，不同步坐标。** 因为每个玩家的家具摆放不同（`PlacementManager` 按玩家独立存档），好友只广播「我想坐下」这个意图枚举，每个客户端用**自己本地的家具锚点**去实现这个姿势。免去布局一致性死结、防穿模作弊、带宽极低（一个整数）。
4. **高度复用现有架构。** `RemoteAvatar` 复用 customer/pet 的 `Node2D + Sprite2D + Area2D + apply_data()` 组合；`SteamManager` 照 6 个子管理器范式做第 7 个 autoload；聊天面板照 wardrobe/inspiration_panel 覆盖层范式；Godot-Global-Input 捕获的全局键鼠可直接当同步主数据。

## 二、整体架构

```
┌─────────────────────────────────────────────────────────┐
│  表现层（本地渲染）                                        │
│  RemoteAvatar × N  ← MultiplayerSpawner 自动复制          │
│  AnchorSpot / AnchorManager（家园本地锚点）               │
│  ChatPanel 覆盖层 + 头顶气泡 SpeechBubble                 │
├─────────────────────────────────────────────────────────┤
│  同步层                                                   │
│  SteamMultiplayerPeer（server_relay=true）               │
│  @rpc：set_intent(pose) / say(text) / position           │
├─────────────────────────────────────────────────────────┤
│  网络层（第 7 个 autoload 子管理器）                       │
│  SteamManager：init / 大厅 / 邀请 / 回调 / 降级           │
│  ↕ Steam 大厅（FRIENDS_ONLY, max 100）+ Valve 中继        │
└─────────────────────────────────────────────────────────┘
```

### 拓扑要点
- `server_relay = true`：客户端间不直连，全经 host 中转 → 拓扑简单、支持中途加入。
- `allowP2PPacketRelay(true)`：NAT 穿透失败时自动回落 Valve 中继。
- 无 Steam 环境自动降级为单机，现有玩法完全不受影响。

## 三、前置条件（硬依赖，不解决无法联机）

| # | 前置 | 说明 |
|---|---|---|
| 1 | **Steamworks App ID** | 后台注册游戏拿 App ID（一次性 $100）。开发期可用测试号 `480`（Spacewar）联调，但正式发布必须自己的 App ID。 |
| 2 | **GodotSteam 插件** | Godot 4.x 专用 Steamworks 绑定，与现有 GDExtension 并存无冲突。装好后 `steam_init` + 回调轮询。 |
| 3 | **Steam 运行检测** | 启动检查 Steam 在跑 + 玩家已登录；否则降级单机。 |

## 四、数据模型与关键组件

### 1. SteamManager（第 7 个 autoload 子管理器）

```gdscript
# scripts/steam_manager.gd
extends Node
# Steam 初始化 + 回调轮询（_process 里 Steam.run_callbacks()）
# 大厅：create_lobby(FRIENDS_ONLY, max) / join_lobby / leave_lobby / invite_friend
# 联网：SteamMultiplayerPeer create_host / create_client（server_relay=true）
# allow_p2p_packet_relay(true)
signal friend_connected(steam_id: int, name: String)
signal friend_disconnected(steam_id: int)
signal chat_received(steam_id: int, msg: String)
signal lobby_ready
```
存档（如需）走 `user://` + `Utils.SAVE_VERSION`，与现有存档框架一致。

### 2. AnchorSpot（锚点节点，放家园场景）

```gdscript
# scripts/anchor_spot.gd
class_name AnchorSpot extends Node2D
enum Type { SIT, CASHIER, BROWSE_RACK, IDLE }
@export var type: Type = Type.SIT
@export var pose_region: Rect2 = Rect2()        # 该姿势对应的贴图裁剪区
@export var facing: int = 1                      # 朝向（flip_h）
@export var nav_offset: Vector2 = Vector2.ZERO   # 走到锚点附近的落脚点
var occupant: int = 0                            # 0=空闲，否则=peer_id
func is_free() -> bool: return occupant == 0
```
v1 在 `main.tscn` 家园层手动放：椅子旁 SIT、展架旁 BROWSE_RACK、收银台 CASHIER。后期让摆件自身派生 AnchorSpot，随家具移动。

### 3. AnchorManager（家园本地，非 autoload）

```gdscript
# scripts/anchor_manager.gd
class_name AnchorManager extends Node2D
@export var spots: Array[AnchorSpot] = []
func claim(type: int, peer_id: int) -> AnchorSpot:
    for s in spots:
        if s.type == type and s.is_free():
            s.occupant = peer_id
            return s
    return null                 # 满了 → 返回 null，化身降级闲逛
func release(anchor: AnchorSpot, peer_id: int) -> void:
    if anchor != null and anchor.occupant == peer_id:
        anchor.occupant = 0
```

### 4. RemoteAvatar（MultiplayerSpawner 复制，复用 customer/pet 组合）

```gdscript
# scripts/remote_avatar.gd
class_name RemoteAvatar extends Node2D
enum State { WANDERING, WALKING, AT_ANCHOR }
var _peer_id: int
var _anchor: AnchorSpot = null
@onready var _sprite: Sprite2D = $Sprite2D
@onready var _animator: AnimationPlayer = $AnimationPlayer

## 拥有者调用，广播意图到所有客户端（含自己）
@rpc("any_peer", "call_local", "reliable")
func set_intent(pose_type: int) -> void:
    _release_anchor()
    var am: AnchorManager = get_tree().get_first_node_in_group("anchor_manager")
    _anchor = am.claim(pose_type, _peer_id)
    if _anchor != null:
        _walk_to(_anchor.global_position + _anchor.nav_offset)
    else:
        _start_wander()          # 没空位 → 闲逛

## 聊天：拥有者发消息，各客户端在化身头顶弹气泡
@rpc("any_peer", "call_local", "reliable")
func say(text: String) -> void:
    _show_bubble(text)

func _on_arrived() -> void:
    if _anchor != null:
        _sprite.region_rect = _anchor.pose_region    # 切到坐/站姿势贴图
        _sprite.flip_h = _anchor.facing < 0
        _animator.play("idle_pose")                   # 轻微呼吸/晃动
```

拥有者触发（在好友自己客户端里点动作菜单）：
```gdscript
avatar.set_intent.rpc(AnchorSpot.Type.SIT)   # 坐下
avatar.say.rpc("今天好累啊")                  # 说话
```

## 五、姿势 / 动作素材

美术沿用 736×414 共享画布 + region 裁切，每个姿势 = 一段 region：

| 姿势 | 做法 |
|---|---|
| **SIT** | 坐姿小人 region（或复用站姿 + AnimationPlayer 做「下蹲」过渡） |
| **CASHIER** | 站姿 + 手抬起 region |
| **BROWSE_RACK** | 侧身看展架 region |
| **IDLE / WANDER** | 现有站姿 + 行走 bob（pet 同款 `sin` 抖动） |

建议新增 `char_poses.png`（同尺寸画布并排画各姿势），或在原画布扩展。每个姿势用 AnimationPlayer 加 2~3 帧呼吸循环，避免僵硬。

## 六、聊天方案（两种可叠加）

| 方案 | 实现 | 适合 |
|---|---|---|
| **Steam 大厅聊天**（原生） | `send_lobby_chat_msg` / `lobby_chat_msg` 回调，文本 ≤4KB，Valve 自带过滤 | 正经长对话面板，开箱即用 |
| **游戏内气泡**（自定义 RPC，推荐） | `@rpc("any_peer","reliable")` 广播 → 对方化身头顶弹气泡 → 淡出 | 沉浸式「好友在你桌面说话」，Bongo Cat 味 |

建议两者都做。自定义 RPC 气泡需自加敏感词过滤（发布合规）。气泡挂 `RemoteAvatar` 头顶跟随位置；多人同坐一排椅子聊天 = 一排小人头顶冒泡。

## 七、分步实现步骤

> 分两大块：**联机基础（P0–P5）** 与 **家园动作系统（A–F）**。二者叠加：先有 Steam 联网，再加锚点动作。
> **A 阶段不依赖 Steam，可单机先行，不阻塞。**

### 第一块：Steam 联机基础

| 阶段 | 内容 | 产出 / 验收 |
|---|---|---|
| **P0 前置** | Steamworks 注册拿 App ID；装 GodotSteam；`SteamManager` 骨架 + 初始化 + 运行检测 + 无 Steam 降级 | 启动能连上 Steam |
| **P1 大厅** | `create_lobby(FRIENDS_ONLY)` / `join_lobby` / 好友邀请 / 大厅码；`lobby_created/joined` 回调 | 能拉好友进同一大厅 |
| **P2 联网** | `SteamMultiplayerPeer` host/client + `server_relay=true`；`MultiplayerSpawner` 在家园 spawn `RemoteAvatar` | 好友化身出现在你桌面 |
| **P3 状态同步** | `@rpc` 同步化身 position / 动作；接入现有 `get_keys_pressed_detailed()` 步数广播 | 看到好友化身跟着对方键鼠动 |
| **P4 聊天** | RPC 气泡 + Steam 大厅聊天面板；`chat_received` 信号 | 能互发消息 |
| **P5 打磨** | 化身装扮同步（复用 ClothesData/换装）、表情、掉线重连、人数上限 | 联机体验完整 |

### 第二块：家园动作系统（意图同步 + 本地锚点）

| 阶段 | 内容 | 产出 / 验收 |
|---|---|---|
| **A 锚点系统** | `AnchorSpot` + `AnchorManager`；家园放 3~4 个锚点；本地测试化身走过去播放姿势 | **单机就能坐/站**（不依赖 Steam） |
| **B 姿势贴图** | 扩展 `char_poses.png` 或 region；AnimationPlayer 呼吸循环 | 坐/站/看展架有图 |
| **C 意图同步** | `set_intent` RPC + MultiplayerSpawner；本地投射逻辑 | 联机时好友能坐你家椅子 |
| **D 动作菜单** | 好友点自己化身弹「坐下/站收银台/看展架/闲逛」菜单 | 玩家可选动作 |
| **E 聊天气泡** | `say` RPC + 气泡 UI + 敏感词过滤 | 坐着能聊天 |
| **F 打磨** | 多人同锚点（多把椅子）、锚点满降级、离线释放、顾客统一改锚点 | 体验完整 |

### 推荐推进顺序
1. **先做 A（单机锚点）** → 立即可见效果，不阻塞、不依赖 App ID。
2. 并行申请 **Steamworks App ID**（硬前置）。
3. App ID 到位后做 **P0→P1→P2**，把化身搬进联机。
4. **C** 接上意图同步，**B/D/E** 补素材与交互，**P4/F** 收尾。

## 八、架构红利：统一「会找锚点的角色」

现有**顾客系统**已在做类似的事（`_decide_behavior` 走向展架购买）。把顾客也改用 `AnchorSpot`（BROWSE_RACK），则顾客 / 好友 / 宠物 / 本地玩家统一为「会找锚点的角色」。可选重构，但能让锚点系统一鱼多吃。

## 九、待拍板决策点

| # | 决策 | 建议 |
|---|---|---|
| 1 | Steamworks App ID 何时申请 | **尽早**——硬前置，无它只能用 480 联调，无法正式发布 |
| 2 | 联机定位：纯环境共享 vs 好友进「我的家园」互动 | 本方案取**后者**（更游戏化） |
| 3 | host 模型：房主制 vs 任意开厅 | **房主制**（简单、匹配放置游戏） |
| 4 | 同步粒度：仅动作+气泡 vs 含游戏状态 | **强烈仅前者**——状态同步引发一致性/作弊爆炸 |
| 5 | 锚点放法：固定场景锚点 vs 家具派生 | **先固定后派生** |
| 6 | 好友选动作：动作菜单 vs 拖到锚点 vs 纯自动 | **菜单 + 轻自动** |
| 7 | 姿势素材：新画 region vs 程序化变形 | **新画**（治愈游戏颜值即正义） |
| 8 | 本地玩家也能坐吗 | 能则一致性好，需加本地锚点交互 |
| 9 | 同类型多锚点 | **多锚点**（2~3 把椅子，多人同坐更有氛围） |
| 10 | 聊天审核 | Steam 大厅走 Valve 过滤；自定义 RPC 气泡自加敏感词过滤 |

## 十、遵循的工程规范

- 组合优于继承、低耦合高内聚、SRP/DI/OCP、信号解耦、autoload 最小化
- 数据-外观分离：Resource 只管「是什么」，场景管「长什么样」
- 存档统一走 `user://` + `Utils.SAVE_VERSION`（为 Steam Cloud 同步预备）
- `SteamManager` 照现有 6 子管理器范式做第 7 个 autoload，不引入新范式
